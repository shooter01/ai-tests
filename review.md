## Summary
В этом PR добавлены проверки на наличие `enterpriseID` и `userID`, а также изменена обработка ошибки при отсутствии члена команды в функции `DeleteEnterpriseMember`.

## Findings
- [medium] services/enterprise/delete.go — Добавление строки `return fmt.Errorf("enterprise id is required")` может привести к утечке информации о структуре кода. Лучше использовать более нейтральный текст или ошибку, не указывающую на конкретные параметры.
- [low] routers/api/enterprise.go — Проверка `if enterpriseID <= 0 || userID <= 0` в функции `DeleteEnterpriseMember` может быть слишком строгой. В зависимости от бизнес-логики, возможно, стоит добавить логирование или более详细的 обработку ошибок.

## Suggestions
1. Используйте нейтральный текст вместо конкретных параметров в сообщениях об ошибках.
2. Добавьте комментарии к новым проверкам для объяснения их назначения и логики.
3. Рассмотрите возможность использования более детальной обработки ошибок в API, например, с кодами состояния HTTP или специальными объектами ошибок.
4. Проверьте, не будет ли эта реализация влиять на другие частями системы, которые могут вызывать эту функцию.
5. Убедитесь, что все тесты прошли после изменения и добавьте дополнительные тесты для новых проверок.
